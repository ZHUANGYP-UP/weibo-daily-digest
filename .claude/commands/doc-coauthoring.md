# 文档协作撰写工作流

引导用户完成协作文档创建，经历三个阶段：上下文收集、完善与结构化、读者测试。

## 阶段 1：上下文收集

1. 询问文档类型、主要受众、期望影响、格式约束
2. 鼓励用户倾倒所有上下文（不用管组织）
3. 提出澄清问题（5-10 个），填补理解空白
4. 退出条件：当可以询问边缘案例而不需要解释基础知识时

## 阶段 2：完善与结构化

逐节构建文档。对每个章节：
1. 提出关于应包含什么内容的澄清问题
2. 头脑风暴 5-20 个选项
3. 用户指示保留/删除/合并
4. 起草该章节
5. 通过精确编辑进行完善

从未知最多的章节开始。连续 3 次迭代没有实质性更改时，询问是否可以删减。

接近完成时（80%+），重新通读检查流畅性、冗余、矛盾。

## 阶段 3：读者测试

1. 预测读者问题（5-10 个）
2. 用子代理（无上下文泄露）测试文档
3. 检查歧义、错误假设、矛盾
4. 发现问题则回到完善阶段修复

## 技巧

- 直接且程序化的语气
- 始终给用户调整流程的主动权
- 质量优于速度，每次迭代都应有有意义的改进
