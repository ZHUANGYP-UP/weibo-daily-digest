# API 路由约束

适用于 `app/api/` 目录下所有路由处理器。

## 响应格式

- 统一使用 `NextResponse.json()` 返回 JSON
- 分页接口返回 `{ data, total }` 格式
- 单条查询返回对象本身
- 删除/更新操作返回 `{ success: true }` 或更新后的对象
- POST 成功返回 `status: 201`

## 错误处理

- **所有路由必须 try-catch 包裹异步操作**
- 验证失败返回 `400` + `{ error: "中文错误信息" }`
- 资源不存在返回 `404` + `{ error: "中文错误信息" }`
- 多个验证错误用分号合并返回
- 禁止空 catch 块，至少 `console.error`

## 输入校验

- 用户输入必须服务端校验，不信任前端
- 通用校验函数放 `lib/validation.ts`
- 字符串输入必须 `trim()` 后再使用
- 校验逻辑在 Prisma 操作之前执行

## 路由参数

- 未使用的请求参数命名为 `_req`
- 动态路由参数通过 `{ params }` 解构获取
- 每个处理器必须有中文 JSDoc 说明用途和 HTTP 方法
